Overview:

This projects a automatic tool that validates globally installed npm packages, checks for available updates, and upgrades them sequentially to ensure stability. It also verifies whether newer versions of Node.js and Git are available.

Plan:
1. Once this "pnpm run start" script will run, it will execute the "npm outdated -g" command.
2. If there are no outdated packages, it will skip to the next step (See point 65).
3. If there are outdated global packages, it will display #COUNT# outdated packages, updating... (It will be displayed from now on a progress bar with animated loading - See how we implemented it on C:\Or\web\projects\repos-maintainer.
4. One by one (not all together), it will take from the list the name of the outdated package, and will update it. On the progress bar it will be displayed as the following:
#ANIMATED LOADER# Updating create-next-app | 16.2.3 -> 16.2.4 [1/13]
5. If any of the update packages are fail, we will try catch it and continue to the next package update.
6. We will list in the end the packages we failed to update and the specific error we got.
7. This script will be run and execute by a bat file from the desktop, See how we implemented it on C:\Or\web\projects\repos-maintainer - C:\Users\Or Assayag\Desktop\reposScanReporter.bat
8. After finish with all the packages updated, we will run "node --version" and "git --version" and check the versions against the latest version.
9. On the end, we will display a list of all the updated packages:
create-next-app      | 16.2.3     -> 16.2.4
firebase-tools       | 15.14.0    -> 15.16.0
node                 | 24.0.0     -> 25.0.0 ! Needs to be updated manually
git                  | 24.0.0     -> 25.0.0 ! Needs to be updated manually
